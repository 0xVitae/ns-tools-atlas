# Submitting a Project

Any NS member can submit their project to the Atlas. Before you begin, make sure you meet the prerequisites below — they vary depending on what type of project you're submitting.

## Prerequisites (all types)

### 1. Sign in

You must be signed in to submit a project. Authentication is handled through your NS account and Discord.

### 2. Be an NS member

You need to be a verified Network School member. During submission, you'll be asked to verify your NS profile URL to confirm you're part of the community.

:::info Profile URL format
Your profile URL looks like: `https://ns.com/your-username`
:::

### 3. Set up NS Authentication with Discord

The Atlas uses NS Auth to verify your NS identity via Discord. If you haven't set this up yet, the quickest way is to use the **Claude Code plugin** — just run:

```
/ns-auth
```

This skill walks you through the full NS Auth integration step by step, including Discord verification.

For manual setup or more details, see the [Claude Code Plugin docs](/docs/claude-code-plugin).

## Project types

Different projects have different requirements. Pick the type that best describes yours.

### Platform / Tool

**Categories:** Networks, Education, Media, Corporate

A digital product, SaaS, app, or web tool — something with a login, dashboard, or persistent web presence.

| Field               | Required | Notes                                            |
| ------------------- | -------- | ------------------------------------------------ |
| **Name**            | Yes      | Your project or tool name                        |
| **Website URL**     | Yes      | Your app or landing page                         |
| **Category**        | Yes      | Pick from the list, or propose a new one         |
| **Description**     | Yes      | Short description (max 160 chars)                |
| **Emoji or Logo**   | Yes      | Visual identity for the Atlas                    |
| **NS Profile URLs** | Yes      | At least 1 team member's NS profile              |
| **Product Images**  | Yes      | At least 1 screenshot of the platform            |
| **Plans**           | Yes      | At least 1 plan (even if free-tier only)         |
| **Guide URL**       | No       | Link to docs, onboarding, or how-to guide        |

### Events / Physical

**Categories:** Events, Coworking, Transport

A venue, event series, coworking space, or physical service. These don't necessarily have a web platform.

| Field               | Required | Notes                                            |
| ------------------- | -------- | ------------------------------------------------ |
| **Name**            | Yes      | Your project name                                |
| **Website URL**     | Yes      | Your website, booking page, or **calendar link** |
| **Category**        | Yes      | Pick from the list, or propose a new one         |
| **Description**     | Yes      | Short description (max 160 chars)                |
| **Emoji or Logo**   | Yes      | Visual identity for the Atlas                    |
| **NS Profile URLs** | Yes      | At least 1 organizer/owner's NS profile          |
| **Product Images**  | No       | Photos of venue or events (recommended)          |
| **Plans**           | No       | Not required                                     |
| **Guide URL**       | No       | Not required                                     |

:::tip Events — provide a subscribable calendar link
If you run a recurring event or event series, use a **subscribable calendar link** as your Website URL — Google Calendar, Luma, lu.ma, Eventbrite, etc. This lets people subscribe to your upcoming events directly.
:::

### VC / Accelerator

**Categories:** Local VCs, Global VCs, Accelerators

An investment fund, accelerator program, or similar entity.

| Field               | Required | Notes                                            |
| ------------------- | -------- | ------------------------------------------------ |
| **Name**            | Yes      | Fund or program name                             |
| **Website URL**     | Yes      | Portfolio page or application page               |
| **Category**        | Yes      | Pick from the list, or propose a new one         |
| **Description**     | Yes      | Short description (max 160 chars)                |
| **Emoji or Logo**   | Yes      | Visual identity for the Atlas                    |
| **NS Profile URLs** | Yes      | At least 1 partner/team member's NS profile      |
| **Guide URL**       | No       | Link to application process, thesis, or FAQ (recommended) |
| **Product Images**  | No       | Not required                                     |
| **Plans**           | No       | Not required                                     |

## How to submit

1. Open the Atlas at `/`
2. Click the **Add Project** button
3. Check the "I have read the docs" checkbox
4. Fill in your project details and submit
5. Agree to form on ns.com/platform
